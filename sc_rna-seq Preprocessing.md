<h1 id="top"> Pre-processing of 10X Single-Cell RNA-seq Data using Galaxy & STARsolo</h1>

<p>
A complete end-to-end preprocessing workflow that converts raw 10X Genomics FASTQ files into a high-quality filtered gene expression matrix using STARsolo within the Galaxy platform.
This pipeline follows best practices in single-cell transcriptomics and prepares data for downstream analysis such as clustering, UMAP visualization, and cell-type annotation.
</p>

<h2 id="introduction"> Introduction</h2>

<p>
Single-cell RNA sequencing (scRNA-seq) is a powerful technology that enables the study of gene expression at the resolution of individual cells. Unlike bulk RNA sequencing, which averages gene expression across many cells, scRNA-seq reveals cellular heterogeneity and allows identification of rare cell populations.
</p>

<p>
However, raw data generated by 10X Genomics contains significant technical noise such as PCR duplicates, empty droplets, and low-quality cells. Therefore, preprocessing is essential to transform raw sequencing reads into a clean and biologically meaningful gene expression matrix.
</p>


<h2 id="background">Background Concepts</h2>

<h3>🔹 10X Genomics Technology</h3>

<p>
10X Genomics uses a droplet-based microfluidic system where individual cells are encapsulated into droplets along with gel beads containing unique barcodes. Each read contains:
</p>

<ul>
  <li><b>Cell Barcode (CB):</b> Identifies the cell of origin</li>
  <li><b>UMI:</b> Identifies unique RNA molecules and removes PCR duplicates</li>
</ul>

<h3>🔹 STARsolo vs Cell Ranger</h3>

<p>
Two major pipelines exist for preprocessing 10X data. Cell Ranger is the official pipeline but is computationally heavy. STARsolo is a faster alternative integrated into the STAR aligner that performs demultiplexing, alignment, and quantification in one step.
</p>

<h3>🔹 File Formats</h3>

<ul>
  <li><b>FASTQ:</b> Raw sequencing reads (R1 = barcode, R2 = cDNA)</li>
  <li><b>MTX:</b> Sparse gene expression matrix</li>
  <li><b>HDF5:</b> Efficient storage format</li>
</ul>

<h2 id="methodology">Methodology</h2>

<h3>1️⃣ Data Upload</h3>

<p>
FASTQ files, barcode whitelist, and gene annotation files are uploaded into Galaxy. Proper pairing of R1 (barcode) and R2 (cDNA) reads is essential for correct processing.
</p>

<pre><code>
# Example (conceptual)
Upload:
- R1.fastq.gz
- R2.fastq.gz
- whitelist.txt
- annotation.gtf
</code></pre>

<h3>2️⃣ Identify Chemistry</h3>

<p>
The read length in R1 determines the chemistry version. A length of 28 bp corresponds to Chromium v3, where 16 bp is barcode and 12 bp is UMI.
</p>

<pre><code>
# Check read length
zcat sample_R1.fastq.gz | head -n 4
</code></pre>

<h3>3️⃣ STARsolo Processing</h3>

<p>
STARsolo performs demultiplexing, alignment, and quantification. It extracts barcodes and UMIs, aligns reads to the genome, and generates a gene count matrix.
</p>

<pre><code>
STAR --runThreadN 8 \
--genomeDir hg19_index \
--readFilesIn R2.fastq.gz R1.fastq.gz \
--soloType CB_UMI_Simple \
--soloCBlen 16 \
--soloUMIlen 12 \
--soloCBwhitelist whitelist.txt
</code></pre>

<h3>4️⃣ Output Files</h3>

<p>
STARsolo produces multiple output files including alignment logs, BAM files, and count matrices in MEX format.
</p>

<pre><code>
matrix.mtx
barcodes.tsv
features.tsv
</code></pre>

<h3>5️⃣ MultiQC Analysis</h3>

<p>
MultiQC summarizes alignment statistics such as mapping rate and read quality. High mapping rates indicate good sequencing quality.
</p>

<pre><code>
multiqc .
</code></pre>

<h3>6️⃣ Droplet Filtering</h3>

<p>
DropletUtils removes empty droplets and low-quality cells. It uses statistical methods such as DefaultDrops and EmptyDrops to retain real cells.
</p>

<pre><code>
library(DropletUtils)

sce <- read10xCounts("matrix/")
sce <- emptyDrops(counts(sce))
</code></pre>

<h3>7️⃣ Barcode Rank Plot</h3>

<p>
This plot distinguishes real cells from background noise using knee and inflection points.
</p>

<pre><code>
barcodeRanks(counts(sce))
</code></pre>



<h2 id="qc">Quality Control</h2>

<p>
Quality control ensures that only high-quality cells are retained. Key metrics include gene counts, UMI counts, and mitochondrial percentage.
</p>

<ul>
  <li>Low genes → empty droplets</li>
  <li>High UMIs → doublets</li>
  <li>High mitochondrial % → damaged cells</li>
</ul>

<h3 id="cell-filtering-results"> Cell Filtering Results</h3>

<p>
Cell filtering is a critical step in single-cell RNA-seq preprocessing, as raw count matrices contain a large number of empty droplets and low-quality cells. 
To accurately identify real cells, two statistical approaches were applied using the DropletUtils package: <b>DefaultDrops</b> and <b>EmptyDrops</b>.
</p>

<p>
The <b>DefaultDrops</b> method uses heuristic thresholds based on expected cell counts, while <b>EmptyDrops</b> applies a more statistically rigorous approach 
using Monte Carlo simulations to distinguish true cells from background RNA contamination.
</p>

<ul>
  <li><b>DefaultDrops:</b> 272 high-quality cells detected</li>
  <li><b>EmptyDrops:</b> 279 high-quality cells detected</li>
</ul>

<p>
The slightly higher number of cells detected by EmptyDrops indicates improved sensitivity, allowing recovery of borderline cells that may still be biologically relevant.
The barcode rank (knee) plot shows a clear separation between real cells (high UMI counts) and empty droplets (low UMI counts).
</p>

<h4>DropletUtils in R</h4>

<pre><code class="language-r">
# Load library
library(DropletUtils)

# Read 10X matrix (MEX format)
sce <- read10xCounts("filtered_matrix_directory/")

# Extract count matrix
counts_matrix <- counts(sce)

# --- Method 1: DefaultDrops (heuristic filtering) ---
default <- barcodeRanks(counts_matrix)

# Estimate number of real cells
summary(default)

# --- Method 2: EmptyDrops (statistical filtering) ---
set.seed(123)  # reproducibility

empty <- emptyDrops(counts_matrix)

# Filter cells based on FDR threshold
filtered_cells <- which(empty$FDR <= 0.01)

# Number of detected cells
length(filtered_cells)
</code></pre>

<h2 id="limitations"> Limitations</h2>

<ul>
  <li><b>Subsampled dataset (~300 cells):</b> The analysis was performed on a reduced dataset for training purposes, which may limit the detection of rare cell populations and biological diversity.</li>
  
  <li><b>No doublet detection:</b> Doublets (two cells captured as one) were not explicitly identified using tools like Scrublet, which may introduce minor inaccuracies in downstream analysis.</li>
  
  <li><b>No ambient RNA correction:</b> Background RNA contamination (e.g., from empty droplets) was not corrected using tools such as SoupX or CellBender.</li>
  
  <li><b>Reference genome (hg19):</b> The analysis used the older hg19 (GRCh37) genome build; newer versions like hg38 provide improved annotation and gene coverage.</li>
</ul>
